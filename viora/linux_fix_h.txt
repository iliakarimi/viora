بله، **PyAutoGUI روی لینوکس هم کار می‌کند**، اما میزان پایداری آن به محیط گرافیکی شما بستگی دارد.

---

## وضعیت کلی روی لینوکس

### ✅ اگر از X11 استفاده می‌کنی

معمولاً بدون مشکل جدی کار می‌کند.
PyAutoGUI روی لینوکس برای کنترل ماوس و کیبورد به **X11** متکی است (از ابزارهایی مثل `xdotool` و `scrot` استفاده می‌کند).

در این حالت:

* حرکت موس ✔
* کلیک ✔
* تایپ ✔
* اسکرین‌شات ✔
* image recognition ✔ (با نصب Pillow و OpenCV)

---

### ⚠ اگر از Wayland استفاده می‌کنی (پیش‌فرض خیلی از توزیع‌های جدید مثل Fedora)

اینجا مشکل شروع می‌شود.

Wayland به دلایل امنیتی:

* اجازه کنترل مستقیم موس/کیبورد نمی‌دهد
* گرفتن اسکرین‌شات سراسری محدود است
* automation ابزارهای X11 را بلاک می‌کند

در نتیجه PyAutoGUI:

* ممکن است اصلاً کار نکند
* یا فقط بخشی از قابلیت‌ها کار کند

---

## اگر Fedora داری (که قبلاً درباره‌اش صحبت کردیم)

Fedora معمولاً با Wayland بالا می‌آید.
اگر می‌خواهی PyAutoGUI پایدار کار کند:

در صفحه login:

1. روی آیکون چرخ‌دنده کلیک کن
2. گزینه **GNOME on Xorg** را انتخاب کن
3. وارد شو

بعدش PyAutoGUI خیلی پایدارتر خواهد بود.

---

## وابستگی‌های لازم روی لینوکس

معمولاً باید این‌ها نصب باشند:

```bash
sudo dnf install python3-tkinter scrot
```

و در pip:

```bash
pip install pyautogui pillow opencv-python
```

## آیا برای پروژه جدی مناسب است؟

اگر:

* تست اتوماسیون ساده می‌خواهی ✔
* اسکریپت شخصی می‌نویسی ✔

اما اگر:

* automation حرفه‌ای برای GUI لینوکس می‌خواهی
* یا Wayland را نمی‌خواهی تغییر بدهی

گزینه‌های بهتر:

* ابزارهای مخصوص X11 مثل xdotool
* برای تست اپلیکیشن‌ها → Selenium (اگر وب است)
* برای GUI لینوکس → dogtail (برای GNOME)
